# WordPress Project Discovery & Branding

Complete discovery interview and branding process for new WordPress projects.

---

## Phase 1: Client Discovery Interview

Use `AskUserQuestion` tool to gather this information before starting development.

### Business Profile

```markdown
## About Your Business

1. **Company Name**: _______________
2. **Industry/Niche**: _______________
3. **Years in Business**: _______________
4. **Location(s)**: _______________

5. **What does your business do?** (2-3 sentences)
   _______________________________________________

6. **Who is your target audience?**
   - Demographics (age, income, location):
   - Psychographics (interests, values):
   - Pain points they have:

7. **Who are your main competitors?**
   - Competitor 1: _______________
   - Competitor 2: _______________
   - Competitor 3: _______________

8. **What makes you different/better?**
   _______________________________________________
```

## Phase 4: Admin & Handoff

### Client Access

```markdown
## Admin Requirements

1. **Who will manage the site?**
   [ ] Client team (needs training)
   [ ] Agency ongoing management
   [ ] Mix of both

2. **Admin skill level:**
   [ ] Beginner (needs simple interface)
   [ ] Intermediate (comfortable with WordPress)
   [ ] Advanced (can handle complexity)

3. **White-label admin?**
   [ ] Yes - remove WordPress branding
   [ ] No - standard WordPress admin

4. **Custom login URL?**
   [ ] Yes - preferred URL: /_______________
   [ ] No - use /wp-admin

5. **Features to hide from client:**
   [ ] Plugins page
   [ ] Theme editor
   [ ] Advanced settings
   [ ] Comments
   [ ] Users (except their own)
```

### Training & Documentation

```markdown
## Handoff Requirements

1. **Training needed:**
   [ ] Video tutorials
   [ ] Written documentation
   [ ] Live walkthrough session
   [ ] None - client is experienced

2. **Documentation topics:**
   [ ] How to edit pages
   [ ] How to add blog posts
   [ ] How to manage [custom post type]
   [ ] SEO best practices
   [ ] Image optimization
   [ ] Form management

3. **Ongoing support:**
   [ ] Included in project
   [ ] Separate maintenance contract
   [ ] As-needed basis
   [ ] Not required
```

## Using Discovery with Claude

### AskUserQuestion Integration

When starting a new project, use the AskUserQuestion tool:

```javascript
// Example: Initial discovery question
AskUserQuestion({
  questions: [
    {
      question: "What type of WordPress project is this?",
      header: "Project Type",
      options: [
        { label: "New site build", description: "Starting from scratch with a new design" },
        { label: "Site redesign", description: "Existing site needs a fresh look" },
        { label: "Site optimization", description: "Improve performance, SEO, or features" },
        { label: "Specific feature", description: "Add a particular feature or fix" }
      ],
      multiSelect: false
    }
  ]
})
```

### Brand Extraction Script

If client has existing site, extract brand data:

```bash
# Use brand-guide skill
python3 /root/.claude/skills/brand-guide/extract-brand.py \
  --url https://existing-site.com \
  --output ./brand-analysis.md

# Or from theme files
python3 /root/.claude/skills/brand-guide/extract-brand.py \
  --theme-path /path/to/theme \
  --output ./brand-guide.md
```
